Unregister node for shutdown APIedit

Signals that a node should no longer be prepared to shut down.

Requestedit

DELETE _nodes/<node-id>/shutdown

Prerequisitesedit

  • If the Elasticsearch security features are enabled, you must have the manage cluster privilege to use this API.
  • If the operator privileges feature is enabled, only operator users can use this API.

Descriptionedit

Signals that the given node should no longer be prepared for shutdown and can resume operations as a normal node in the cluster.

This API must be used if a node returns to the cluster after the node has been registered for shutdown - requests will not be automatically cleared for either RESTART or REMOVE request types.

Path parametersedit

<node-id>
(Optional, string) The ID of a node which is preparing to be shut down.

Query parametersedit

master_timeout
(Optional, time units) Period to wait for a connection to the master node. If no response is received before the timeout expires, the request fails and returns an error. Defaults to 30s.
timeout
(Optional, time units) Period to wait for a response. If no response is received before the timeout expires, the request fails and returns an error. Defaults to 30s.

This API is only useful after a node has been registered for shutdown:

PUT /_nodes/USpTGYaBSIKbgSUJR2Z9lg/shutdown
{
  "type": "restart",
  "reason": "demonstrating how the node shutdown API works"
}

Once a node is in the process of preparing for shutdown, we can cancel the process by issuing a request like so:

DELETE /_nodes/USpTGYaBSIKbgSUJR2Z9lg/shutdown

Which returns the following response:

{
    "acknowledged": true
}

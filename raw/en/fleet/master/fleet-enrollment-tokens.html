Fleet enrollment tokensedit

An enrollment token is an Elasticsearch API key that you use to enroll one or more Elastic Agents in Fleet. The enrollment token is used for the initial communication between Elastic Agent and Fleet Server. You can use the token as many times as required. It will remain valid until you revoke it.

After the initial connection request from the Elastic Agent to the Fleet Server, Fleet Server passes two API keys to the Elastic Agent:

  • An output API key.

    This API key is used to send data to Elasticsearch. It has the minimal permissions needed to ingest all the data specified by the policy. If the API key is invalid, the Elastic Agent stops ingesting data into Elasticsearch.

  • A communication API.

    This API key is used to communicate with the Fleet Server. It has only the permissions needed to communicate with the Fleet Server. If the API key is invalid, Fleet Server stops communicating with the Elastic Agent.

Create enrollment tokensedit

Mention the enrollment token shown when you add an agent.

Fleet > Agent > Enrollment tokens. Create enrollment token. Select a policy. Use the token when you enroll an agent in Fleet.

Revoke enrollment tokensedit

Under Fleet > Agent > Enrollment tokens, click the Revoke token icon.

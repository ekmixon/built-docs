What are solutions?edit

We’ve streamlined the setup process for the most common use cases for the Elastic Stack by grouping them into solutions. Solutions are specialized templates pre-configured with sensible defaults and settings. The available solutions are:

  • Elastic Enterprise Search
  • Elastic Observability
  • Elastic Security

If you need something else, you can customize the Elastic Stack for whatever you can dream up, using one of our hardware profiles as a starting point.

Elastic Enterprise Searchedit

This solution provides the capabilities of Elastic Enterprise Search, including:

  • App Search - delivering a pre-tuned search experience in your applications.
  • Workplace Search - delivering a fast and relevant search experience across your organization.

To enable Enterprise Search on Elasticsearch Service, see Enable Enterprise Search.

For the rest of the Enterprise Search documentation, see the Enterprise Search Reference.

Included in this solution:

Amazon Web Services (AWS)
  • Elasticsearch data nodes: Defaults to 1 GB memory x 1 availability zone. Hosted on AWS m5 instances.
  • Enterprise Search nodes: Defaults to 4 GB memory x 2 availability zones. Hosted on AWS m5 instances.
  • Kibana: Defaults to 1 GB memory x 1 availability zone. Hosted on AWS r4 instances.
Google Cloud Platform (GCP)
  • Elasticsearch data nodes: Defaults to 1 GB memory x 1 availability zone. Hosted on custom CPU-optimized GCP instances.
  • Enterprise Search nodes: Defaults to 4 GB memory x 2 availability zones. Hosted on custom CPU-optimized GCP instances.
  • Kibana: Defaults to 1 GB memory x 1 availability zone. Hosted on custom memory-optimized GCP instances.
Microsoft Azure
  • Elasticsearch data nodes: Defaults to 1 GB memory x 1 availability zone. Hosted on Azure D64sv3 instances.
  • Enterprise Search nodes: Defaults to 4 GB memory x 2 availability zones. Hosted on Azure D64v3 instances.
  • Kibana: Defaults to 1 GB memory x 1 availability zone. Hosted on Azure E32sv3 instances.

Elastic Observabilityedit

This solution provides all of the data ingestion capabilities to know your infrastructure, including:

  • APM - Monitor software services and applications in real time.
  • Logs - Ingest and view a variety of logs to identify problems.
  • Metrics - Collect data from multiple sources and set custom alerts.

For the rest of documentation, see the Observability Guide.

Included in this solution:

Amazon Web Services (AWS)
  • Elasticsearch data nodes: Defaults to 8 GB memory x 2 availability zones. Hosted on aws.data.highio.i3 instances.
  • APM: Defaults to 512 MB memory x 1 availability zone. Hosted on AWS r5d instances.
  • Kibana: Defaults to 1 GB memory x 1 availability zone. Hosted on AWS r5d instances.
Google Cloud Platform (GCP)
  • Elasticsearch data nodes: Defaults to 8 GB memory x 2 availability zones. Hosted on custom IO-optimized GCP instances.
  • APM: Defaults to 512 MB memory x 1 availability zone. Hosted on custom CPU-optimized GCP instances.
  • Kibana: Defaults to 1 GB memory x 1 availability zone. Hosted on custom memory-optimized GCP instances.
Microsoft Azure
  • Elasticsearch data nodes: Defaults to 8 GB memory x 1 availability zone. Hosted on Azure L32sv2 instances.
  • APM: Defaults to 512 MB memory x 1 availability zone. Hosted on Azure E32sv3 instances.
  • Kibana: Defaults to 1 GB memory x 1 availability zone. Hosted on Azure E32sv3 instances.

Elastic Securityedit

This solution combines SIEM threat detection features with endpoint prevention and response capabilities, including:

  • SIEM - Detection of security threats and event monitoring across your environment.
  • Endpoint Security - Prevention and threat response for your endpoints.

For the rest of documentation, see the Elastic Security Guide.
